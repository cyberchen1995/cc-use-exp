# AGENTS.md

## 工作方式

- 先阅读相关代码、配置和脚本，再下结论。
- 优先做能解决问题的最小改动。
- 优先复用现有模式，再考虑新增抽象。
- 前后端联调出现列表、分页、筛选或详情异常时，先核对接口输入输出契约和项目统一响应格式，再决定是否保留临时兼容。
- 不主动扩需求；只有在安全完成当前任务必须时才扩大改动范围。

## 质量要求

- 将错误行为、类型问题、缺失校验、缺失错误处理、明显回归视为真实问题。
- 将接口输入输出契约漂移（成功响应包装、分页结构、枚举或筛选项结构不一致）视为真实问题，不默认以前端兼容长期掩盖。
- 不通过修改测试来掩盖错误实现。
- 改动后做受影响范围内最小但有意义的验证。

## 安全

- 对破坏性、不可逆或环境敏感操作，先说明风险与影响范围。
- 优先选择可回滚、可验证、影响面更小的步骤。
- 不确定时先收集更多上下文，再执行变更。

## 交互

- 默认保持专业、直接、低噪音表达。
- 默认使用简体中文回复；代码、命令、日志、报错和协议字段保持原文，只有用户明确要求英文或外部材料必须英文时才切换。
- 文档和代码中的作者署名统一使用 `wwj`；如果需要补充作者信息，不使用其他署名。
- Java 代码文件在新增或补齐文件头注释时，头部作者注释统一使用 `@author wwj`。

## 分层

- 语言和框架规范优先参考项目内现有配置目录，例如 `.codex/skills/`、`.claude/skills/`、`.cursor/skills/`、`.github/instructions/`。
- fix、review、design、requirement、commit-msg、optimize、status、new-feature、project-init 这类流程优先复用项目内既有 workflow。
- 审批和越权命令控制交给对应工具的 rules / policies / instructions 体系。
