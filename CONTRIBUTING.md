# Contribution Guide

感谢你为本项目提交改进。

本项目公开版本采用 `PolyForm Noncommercial 1.0.0` 发布。这是 `source-available` 非商用许可，不是 OSI 开源许可。维护者保留公开版本、商业版本和私有版本的完整发布与授权空间。为了避免后续权利链不清，请在提交贡献前阅读并确认以下条款。

如维护者要求，贡献者应在合并前单独签署 [CLA.md](./CLA.md)。未签署前，维护者可以拒绝合并或仅保留沟通记录而不采纳贡献。

默认情况下，未在 `.github/cla/allowlist.txt` 中的贡献者，其 PR 需要由维护者在确认已收到并归档签署版 `CLA` 后手动添加 `cla-signed` 标签，相关执行规则见 [.github/cla/README.md](./.github/cla/README.md)。

## 提交前

- 先阅读 [README.md](./README.md) 中的项目定位、使用方式和许可说明。
- 优先提交最小、清晰、可验证的改动。
- 不要提交你无权授权的代码、文档、图片、字体、数据或其他第三方材料。
- 若你的改动来自其他项目，请在提交说明中明确来源、原许可证及兼容性判断。

## 提交方式

- 通过 Issue 讨论 Bug、功能建议和文档修正方向。
- 通过 Pull Request 提交代码或文档变更。
- 提交说明请尽量写清楚变更原因、范围和验证方式。
- 维护者保留自行决定是否合并、修改、延后、私有化或商业化使用任何贡献的权利。

## 贡献授权条款

向本仓库提交任何 Issue 附带补丁、Pull Request、直接 commit、文档内容或其他可合并的贡献时，除非你在提交前与维护者另有书面约定，否则视为你确认并同意：

1. 你有权提交该贡献，并有权按本文件约定授予相应许可。
2. 你的贡献可以由维护者决定是否进入本项目的公开版本，并可按维护者选择的公开许可证或许可策略发布、再发布或不发布。
3. 你同时授予本项目维护者一项永久的、全球范围的、非独占的、不可撤销的、免版税的许可，允许维护者将你的贡献作为本项目的一部分，以当前或未来的 source-available 许可证、商业许可证、专有许可证或其他许可证进行使用、修改、再许可、分发、销售和利用。
4. 维护者可以在为本项目进行开发、发布、合规、商业授权、私有化部署、品牌管理或版本管理所必需的范围内，对你的贡献进行修改、整理、组合、拆分、再分发或不予发布。
5. 你的贡献不会因为被维护者并入商业版本、私有版本、未公开版本或未来改用其他许可证而产生额外对价、分成、署名承诺或发布义务，除非双方另有书面约定。
6. 如果你不接受上述条款，请不要向本仓库提交贡献。

上述条款旨在让维护者对公开版、商业版和私有版都保持清晰且可执行的权利来源。它不是法律意见；如你需要单独约定，请在提交前联系维护者。

对外部非琐碎贡献，维护者可要求以 [CLA.md](./CLA.md) 作为最终有效的补充授权文件；仓库中的默认条款不妨碍维护者要求更正式的单独签署流程。

## 第三方材料与许可证边界

- 引入第三方代码前，请确认其许可证不会阻断本项目的商业授权、专有版本、后续换证或再许可。
- 未经维护者书面确认，不要提交 `GPL`、`AGPL`、`LGPL`、`MPL`、`EPL`、CC 共享相同方式或其他带 copyleft / reciprocal / field-of-use 限制的代码、文档、素材或模板。
- 对于 `Apache-2.0`、`MIT`、`BSD` 等材料，也要逐项核对版权声明、NOTICE 要求、专利条款和再授权兼容性。
- 不要直接复制来源不清、许可证缺失或明确限制再授权的内容。
