# 文件行数限制

> 控制单文件复杂度，减少 token 消耗，提升可维护性。

---

## 行数上限（默认阈值）

| 语言 | 上限 | 说明 |
|------|------|------|
| Java | 300 行 | Spring 分层清晰，超过说明职责过多 |
| Go | 400 行 | 错误处理导致偏长，但超过 400 应拆分 |
| Vue | 200 行 | 样式已抽到 assets/styles，模板+逻辑 200 行足够 |
| TSX/JSX | 200 行 | 同 Vue |
| TypeScript/JS | 300 行 | 工具类、服务类 |
| Python | 300 行 | 模块化拆分 |

**项目级覆盖**：可在项目 `GEMINI.md` 中定义 `## 文件行数限制` 章节覆盖默认值。

---

## 实时警告机制

### 触发时机

修改代码文件（`.go`、`.java`、`.vue`、`.tsx`、`.jsx`、`.ts`、`.js`、`.py`）时，自动检查行数。

### 检查逻辑

1. 检测文件类型 → 查找对应阈值（优先项目级配置，否则使用默认值）
2. 统计当前文件行数（排除空行和纯注释行）
3. 如果超限：
   - 显示警告信息（文件路径、当前行数、阈值、超限量、拆分建议）
   - 使用交互式问答提供选项：
     - **继续（暂不处理）**：保持当前状态，稍后手动处理
     - **简化（自动拆分）**：调用 `size-check` skill 自动拆分文件
4. 如果未超限：静默通过

### 警告格式

```
⚠️ 文件行数超限

文件：src/services/UserService.java
当前：350 行
阈值：300 行
超限：+50 行

建议：Service 拆分职责、提取 Helper/Converter 类

是否立即简化？
```

### 防重复警告

同一文件在同一会话中仅警告一次，避免频繁打断开发流程。

---

## 超限时的处理

**新建文件时**：预估超限则提前拆分，不要写完再拆

**修改现有文件时**：
- 不强制重构现有大文件
- 新增功能如果会导致超限，拆到新文件
- 触发警告时，用户可选择立即简化或稍后处理

---

## 拆分指引

| 语言 | 拆分方式 |
|------|---------|
| Java | Service 拆分职责、提取 Helper/Converter 类 |
| Go | 按功能拆分同包文件（如 `user_query.go`、`user_update.go`） |
| Vue | 提取子组件、composables、样式到 assets/styles |
| TSX/JSX | 提取子组件、hooks、utils |
| TS/JS | 提取工具函数、常量、类型到独立文件 |
| Python | 按职责拆分模块、提取 utils/helpers |

---

## 完整审查

需要全项目扫描或完整代码简化审查时，使用 `/fix` 命令并描述"简化代码"或"检查文件大小"。详见 `skills/size-check/`。
