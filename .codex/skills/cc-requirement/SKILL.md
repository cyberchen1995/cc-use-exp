# Requirement

当用户明确要求 requirement、requirement doc 或 interrogate 时，使用本技能。

不要用于：

- 技术实现设计
- bug 修复
- 正式 code review
- 语言实现细节

## 核心方式

1. 先明确业务目标和成功标准。
2. 找出模糊点、边界条件和例外场景。
3. 需要整理文档时使用 requirement 模板。
4. 需要高压盘问时使用 interrogate 清单。
